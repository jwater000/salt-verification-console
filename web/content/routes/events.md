# Events

이 페이지는 `사건 단위 관측`이 웹 콘솔 안에서 어떤 의미를 갖는지 설명한다. 책에서 중력파, 초기 우주, 강중력 경로, 다중 검출 정합을 폭넓게 다루더라도, 웹은 그중 `현재 데이터 구조로 재현 가능한 사건형 채널`만 취급한다.

## 기준이 되는 장

- `05장`: 중력파 관측 자체와 파형/도달시간 해석의 구분
- `06장`: 초기 우주 흔적과 파동 채널을 어떻게 읽을지의 문제의식
- `19장`: GR과의 저에너지 정합, 관측 축과 해석 축의 분리
- `27장`: 실제로 운영 잠금할 차별 예측 채널과 반증 기준

## 사건 페이지를 읽는 방법

1. 먼저 `actual_value`가 무엇인지 본다.
2. 같은 이벤트에 대해 `standard_fit`과 `salt_fit`이 어떤 차이를 내는지 본다.
3. 승패보다 먼저 이벤트 품질과 데이터 출처를 확인한다.
4. 독립 데이터셋에서 같은 방향성이 반복되는지 확인한다.

## 책의 설명과 웹의 사건 구조는 어떻게 다른가

| 구분 | 책의 역할 | 웹의 역할 |
| :--- | :--- | :--- |
| 중력파/우주 사건 | 왜 이런 사건이 중요하고 어떤 해석이 가능한지 설명 | 실제 이벤트 row와 residual을 공개 |
| SALT 해석 | 장력 요동, 경로 밀도, 추가 지연 같은 해석 프레임 제시 | 그 해석이 score를 줄 수 있는 채널만 채택 |
| 반증 기준 | 어느 관측에서 틀릴 수 있는지 서술 | dataset, fit, residual, hash 기준으로 운영 잠금 |

## 현재 웹에서 다루는 사건형 채널

- 시간지연/적색편이 계열
- 고에너지 tail 비교
- 실시간 이벤트 snapshot에 연결된 일부 거시 관측

아래 항목은 책에는 나오지만 현재 사건 페이지의 직접 채점 대상은 아니다.

- 초고주파 중력파 꼬리의 독립 이벤트 score
- 편광/맵 기반 초기 우주 채널
- 강중력 렌즈 지연의 대규모 통합 residual

## 왜 이렇게 좁게 잡는가

이벤트 페이지는 설명용이 아니라 `재현 가능한 판정 단위`를 보여주는 공간이기 때문이다. 따라서 사건이 중요하다는 서술만으로는 올라오지 않고, 아래가 갖춰질 때만 편입한다.

1. 공개 데이터셋
2. 이벤트별 비교식
3. 품질 규칙
4. score 또는 residual 정의

그 전 단계의 가설은 `/predictions`에 남겨 두는 것이 현재 웹 구조에 맞다.
