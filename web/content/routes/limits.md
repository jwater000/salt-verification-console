# Limits

이 페이지는 SALT가 어디까지 말할 수 있고 어디서부터는 아직 못 말하는지를 정리한다. 책이 커지면서 설명 범위는 넓어졌지만, 웹은 그중 `현재 판정 가능 범위`와 `아직 남은 공백`을 분리해서 보여줘야 한다.

## 현재 책 구조에서 한계 판단의 기준 장

- `18장`: 양자 코어만으로는 스핀 1/2, 카이랄리티, 디랙 구조를 끝까지 설명하지 못한다는 한계
- `20장`: 예측이 중요한 이유와 검증 가능한 채널의 의미
- `25장`: 식은 제시됐지만 운영형 예측 파이프라인이 아직 부분 구현인 영역
- `27장`: 실제로 잠근 차별 예측은 3개 채널뿐이라는 운영 기준

## 웹이 현재 명시해야 하는 한계

| 항목 | 현재 상태 | 왜 한계로 남는가 |
| :--- | :--- | :--- |
| 양자 코어 -> 스피너/디랙 확장 | 미완성 | 17장은 비상대론 코어, 18장은 확장 조건 정리 단계 |
| GR 정합의 완전 증명 | 미완성 | 19장 개요와 25장 기술식은 있으나, 26장 기준으로도 일부는 잠정 통과 |
| 차별 예측의 운영 잠금 | 제한적 | 27장은 세 채널만 잠금, 나머지는 설명용 가설에 가깝다 |
| 웹 구현 범위 | 부분 구현 | 공개 데이터, score, residual, hash가 있는 채널만 결과화 가능 |

## 특히 구분해야 하는 세 가지

1. `설명 가능성`과 `판정 가능성`은 다르다.
2. `기술식 제시`와 `운영 잠금`은 다르다.
3. `책의 서술`과 `웹의 결과 보고`는 다르다.

## 지금 웹에서 보수적으로 남겨야 하는 문장

- SALT가 모든 양자/상대론 문제를 이미 닫았다고 말하지 않는다.
- GR을 대체하는 완결된 독립 장방정식을 이미 확보했다고 말하지 않는다.
- 아직 score가 없는 예측을 결과 페이지에 섞지 않는다.

## 그래서 현재 웹은 어떻게 정리하는가

- `/evaluation`: frozen 데이터로 직접 채점된 결과만 공개
- `/predictions`: 채널은 있으나 아직 score가 잠기지 않은 가설 공개
- `/engineering`: 20장의 활용 시나리오를 결과와 분리해 설명
- `/audit/reproduce`: 25~27장의 규격과 감사 기준을 재현 관점에서 정리

즉 한계 페이지의 목적은 SALT를 약하게 보이게 하는 것이 아니라, `어디까지가 운영 잠금됐고 어디부터가 아직 연구 프로그램인가`를 흐리지 않게 만드는 데 있다.
