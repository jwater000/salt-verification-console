## 5) 핵심 갭

### Gap 1. 정보구조는 재편됐지만 콘텐츠 모델이 아직 코드에 박혀 있다

현재 상태:

- `Guide`, `Verification`, `Engineering`, `Reference`는 이미 새 구조를 사용
- 하지만 장/채널/용어/FAQ 데이터가 대부분 각 페이지 컴포넌트 내부 상수로 존재

영향:

- 원고가 다시 바뀔 때 웹 동기화 비용이 큼
- 장별 요약과 그래프 메타데이터를 재사용하기 어렵다

### Gap 2. 계획 라우트 트리와 실제 구현 라우트가 아직 1:1 대응이 아니다

현재 빠진 항목:

- `/core/logic-map`
- `/core/chapters`
- `/verification/channels`
- `/verification/candidate-hypotheses`
- 세부 `engineering/*` 하위 페이지

영향:

- 계획 문서 기준으로는 “개념 레퍼런스형 허브”가 아직 완성되지 않음

### Gap 3. 댓글 확장은 기획만 있고 실행 기반이 없다

현재 상태:

- 인증 스택 미선정
- 스키마/환경변수/API/UI 모두 미구현

영향:

- 18장 `Verification`과 19장 `Engineering`에 외부 피드백 루프를 붙이려는 목표가 아직 시작되지 않음

### Gap 4. 도서 원고 정리와 웹 반영 간의 자동 연결 장치가 없다

현재 상태:

- `docs/book/*`는 로컬 전용이며 git 추적 대상이 아님
- 웹 페이지는 수동 편집 결과물에 가깝다

영향:

- 원고 최신판과 웹 요약판의 불일치가 다시 생길 가능성이 높다
