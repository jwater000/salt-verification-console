# Five-Nav Story Wireframes

기준일: `2026-04-06`

이 문서는 [FIVE_NAV_STORY_REENGINEERING_MASTER_PLAN.md](/home/jwater/Development/salt-verification-console/docs/roadmap/current/FIVE_NAV_STORY_REENGINEERING_MASTER_PLAN.md)을 실제 페이지 전개 수준으로 내린 텍스트 와이어프레임 문서다.

대상 내비게이션:

1. `소개`
2. `핵심 아이디어`
3. `내용 검증`
4. `참고자료`
5. `게시판`

상태 메모:

- 주요 허브와 세부 페이지 카피는 이 와이어프레임 방향으로 대부분 반영됐다.
- 남은 차이는 일부 세부 블록 배치 미세조정, CTA 관계 정리, 실제 운영 검증이다.

공통 원칙:

- 첫 화면은 `카드 나열`보다 `문장 리드`를 우선한다.
- 각 페이지는 `리드 -> 전개 -> 정리 -> 근거/다음 이동` 순서를 따른다.
- `요약`, `근거`, `검증식 표현`, `감사 링크`는 후반 블록에 둔다.

## 1. 소개

### 1.1 페이지 역할

- 처음 온 방문자가 SALT를 한 문장으로 이해한다.
- 이 웹이 책의 대체물이 아니라 입문과 길잡이임을 안다.
- 다음에 어디로 가야 할지 바로 판단한다.

### 1.2 전개 순서

#### Hero

- 라벨: `소개`
- 제목:
  - `SALT를 먼저 이해하도록 돕는 웹사이트`
- 리드:
  - `이 사이트는 먼저 SALT가 무엇을 보려는 시도인지 설명하고, 그 다음 핵심 아이디어를 거쳐 내용 검증과 참고자료로 이어지게 설계돼 있다.`
- 서브 문장:
  - `웹은 책 전체를 대신하기보다, 독자가 문제의식과 읽는 순서를 먼저 붙잡도록 돕는 입문 계층이다.`

#### Narrative Scene 1

- 제목: `무슨 이야기를 하려는가`
- 내용:
  - SALT의 한 줄 정의
  - 기존 물리학 설명 공백을 하나의 흐름으로 다시 읽으려는 시도라는 설명

#### Narrative Scene 2

- 제목: `왜 웹이 필요한가`
- 내용:
  - 책은 깊이 있는 논증
  - 웹은 진입, 구조, 검증 위치 파악

#### Narrative Scene 3

- 제목: `어떤 순서로 읽으면 좋은가`
- 내용:
  - `소개 -> 핵심 아이디어 -> 내용 검증 -> 참고자료`

#### Summary Block

- 3~4개 항목 요약
  - 웹의 역할
  - 책과의 관계
  - 추천 읽기 순서

#### Evidence / Next Step

- CTA 1: `핵심 아이디어로 이동`
- CTA 2: `내용 검증으로 이동`
- CTA 3: `책 구매처 보기`

## 2. 핵심 아이디어

### 2.1 페이지 역할

- SALT의 중심 문제의식과 개념 전환을 설명한다.
- 장별 목록보다 논리 흐름을 먼저 보여준다.
- 검증으로 넘어가기 직전까지를 담당한다.

### 2.2 전개 순서

#### Hero

- 라벨: `핵심 아이디어`
- 제목:
  - `SALT의 핵심 아이디어가 어떤 흐름으로 이어지는가`
- 리드:
  - `이 섹션은 장 번호보다 먼저 이야기의 전환점을 보여준다.`
- 서브 문장:
  - `기존 설명에서 무엇이 끊겨 보이는지, SALT가 그 장면들을 어떤 상태변수 언어로 다시 묶는지, 왜 그 해석이 검증 가능한 형태로 이어지는지를 순서대로 읽는다.`

#### Narrative Scene 1

- 제목: `문제`
- 내용:
  - 기존 설명이 분절돼 보이는 이유
  - 공간, 질량, 상호작용을 따로 읽는 한계

#### Narrative Scene 2

- 제목: `단서`
- 내용:
  - 관측 흔적과 상태변수 연결
  - `ρ`, `θ`, `L`이 왜 필요한가

#### Narrative Scene 3

- 제목: `개념 전환`
- 내용:
  - 하나의 상태 변화 문제로 다시 읽는 전환
  - 통일장 해석의 중심 문장

#### Narrative Scene 4

- 제목: `검증으로의 브리지`
- 내용:
  - 이론 설명이 끝나는 지점
  - 비교 가능성이 시작되는 지점

#### Summary Block

- `Problem -> Clue -> Concept -> Solution -> Bridge to Verification`

#### Evidence / Next Step

- CTA 1: `Logic Map 보기`
- CTA 2: `장별 인덱스 보기`
- CTA 3: `내용 검증으로 이동`

## 3. 내용 검증

### 3.1 페이지 역할

- 앞에서 읽은 설명을 실제 비교 자료와 연결해 보는 확인 단계다.
- 결과와 대기 항목, 감사 자료의 위치를 분리해 보여준다.

### 3.2 전개 순서

#### Hero

- 라벨: `내용 검증`
- 제목:
  - `SALT를 이해한 뒤 마지막에 확인하는 내용 검증`
- 리드:
  - `이 섹션은 첫 소개 페이지가 아니라, 앞에서 읽은 아이디어를 실제 비교 자료와 연결해 보는 확인 단계다.`

#### Narrative Scene 1

- 제목: `무엇을 비교하는가`
- 내용:
  - 관측 흔적의 정량 비교
  - 기준선과 SALT의 비교 방식

#### Narrative Scene 2

- 제목: `현재 어디까지 왔는가`
- 내용:
  - 고정 채널
  - 공개 결과
  - 검증 대기 항목

#### Narrative Scene 3

- 제목: `어디까지 따라가 볼 수 있는가`
- 내용:
  - reproduce
  - datasets
  - formulas
  - runs
  - snapshots

#### Summary Block

- 현재 공개 검토 범위
- 결과와 대기 항목 구분
- 감사 자료 위치

#### Evidence / Next Step

- CTA 1: `판정 결과 보기`
- CTA 2: `검증 대기 보기`
- CTA 3: `감사 자료 보기`

## 4. 참고자료

### 4.1 페이지 역할

- 앞에서 읽은 핵심 아이디어를 더 안정적으로 이해하게 돕는다.
- 새로운 주장보다 이해 보조를 담당한다.
- 책과 웹을 왕복하게 한다.

### 4.2 전개 순서

#### Hero

- 라벨: `참고자료`
- 제목:
  - `앞에서 읽은 이야기를 더 안정적으로 붙잡게 하는 참고자료`
- 리드:
  - `이 섹션은 새로운 주장을 늘리기보다 이미 읽은 핵심 아이디어를 더 자연스럽게 이해하게 만드는 보조 계층이다.`

#### Narrative Scene 1

- 제목: `어디서 막히기 쉬운가`
- 내용:
  - 용어
  - 장 대응
  - 오해 지점

#### Narrative Scene 2

- 제목: `책과 웹은 어떻게 대응되는가`
- 내용:
  - Book Map 중심 설명
  - 장과 허브의 관계

#### Narrative Scene 3

- 제목: `어떤 보조 자료를 보면 좋은가`
- 내용:
  - glossary
  - FAQ
  - visual atlas

#### Summary Block

- `이해 보조`
- `책-웹 대응`
- `검증 전 보강 독해`

#### Evidence / Next Step

- CTA 1: `Book Map 보기`
- CTA 2: `FAQ 보기`
- CTA 3: `내용 검증으로 이동`

## 5. 게시판

### 5.1 페이지 역할

- 방문자 질문, 독서 메모, 해석을 주제형으로 모은다.
- 페이지 하단 댓글과 구분되는 별도 대화 공간이다.

### 5.2 전개 순서

#### Hero

- 라벨: `게시판`
- 제목:
  - `방문자 토론과 질문을 모으는 공개 게시판`
- 리드:
  - `이 공간은 페이지별 짧은 댓글보다 조금 더 긴 질문, 해석, 독서 메모를 모아 두는 공개 대화 공간이다.`

#### Narrative Scene 1

- 제목: `여기에는 무엇이 올라오는가`
- 내용:
  - 질문
  - 오해 정리
  - 독서 메모
  - 반론과 해석

#### Narrative Scene 2

- 제목: `페이지 댓글과 어떻게 다른가`
- 내용:
  - 댓글은 특정 페이지 문맥
  - 게시판은 주제형 스레드

#### Narrative Scene 3

- 제목: `운영은 어떻게 이루어지는가`
- 내용:
  - 공개 읽기
  - 인증 쓰기
  - moderation 연결

#### Summary Block

- 게시판 역할
- 댓글과의 차이
- 인증과 운영 정책

#### Evidence / Next Step

- CTA 1: `최신 게시글 보기`
- CTA 2: `로그인 상태 확인`
- CTA 3: `운영 정책 보기`

## 6. 기존 라우트별 1차 와이어프레임 대응

| 새 내비 | 대표 라우트 | 현재 우선 반영 대상 |
| --- | --- | --- |
| `소개` | `/`, `/guide` | 홈 hero, Guide hero, 추천 읽기 순서 |
| `핵심 아이디어` | `/core`, `/core/logic-map` | Core hero, Logic Map 설명 블록 |
| `내용 검증` | `/verification`, `/verification/results`, `/audit` | Verification hero, 결과/대기/감사 연결 |
| `참고자료` | `/reference`, `/reference/book-map` | Reference hero, Book Map 미리보기 |
| `게시판` | `/discussion`, 페이지별 comments, `/audit/comments` | 공개 게시판, 댓글 상태 표시, moderation 연결 |

## 7. 구현 메모

- 1차 구현에서는 기존 라우트를 유지한 채 카피와 블록 순서부터 바꾼다.
- 각 섹션의 `Summary Block`은 현재 카드형 요약을 재활용할 수 있다.
- `Evidence / Next Step` 블록은 기존 `NextSteps`, 링크 카드, audit 링크를 재배치해 구성한다.
