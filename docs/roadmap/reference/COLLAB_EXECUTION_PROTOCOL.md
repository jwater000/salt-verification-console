# Collaboration Execution Protocol

최종 업데이트: `2026-03-06`

## 목적
- 시간 낭비를 줄이고, 요청-실행-검증 사이클을 빠르게 고정한다.
- 불명확한 요청 상태에서 무작정 진행하지 않는다.

## 기본 원칙
1. 작업 시작 전에 목표를 1-2줄로 고정한다.
2. 정보가 부족하면 핵심 질문만 짧게 물어본다.
3. 답을 받으면 바로 실행하고, 중간 진행 상황을 짧게 공유한다.
4. 종료 시 결과와 다음 액션 후보를 1-3개 제시한다.

## 사전 확인 필수 3항목
- 완료 기준: 어디까지 되면 끝인지
- 우선순위/마감: 무엇을 먼저, 언제까지
- 변경 권한 범위: 파일 수정, 명령 실행, 배포/푸시 가능 여부

위 3개 중 하나라도 비어 있으면, 실행 전에 확인 질문을 먼저 한다.

## 질문 규칙
- 질문은 최대 3개 이내로 제한한다.
- 선택지를 강요하지 않고, 의사결정에 필요한 정보만 묻는다.
- 이미 문서/코드에서 확인 가능한 내용은 다시 묻지 않는다.

## 실행 규칙
- 합의된 범위를 벗어나는 변경은 사전 확인 후 진행한다.
- 실행 중 발견된 리스크(일정 지연, 품질 저하, 배포 위험)는 즉시 공유한다.
- 필요 시 더 작은 단위로 쪼개서 우선순위 높은 항목부터 처리한다.

## 완료 보고 규칙
- 무엇을 바꿨는지
- 어디를 바꿨는지(파일 경로)
- 무엇을 검증했는지(빌드/테스트/수동 확인)
- 남은 리스크와 다음 액션

## 커뮤니케이션 톤
- 기본 톤은 직설적이고 간결하게 유지한다.
- 반말/존댓말은 사용자 선호를 우선한다.
